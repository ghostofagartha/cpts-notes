### Nginx - Enabling PUT
- Alternative to **Apache** for transferring files
- Less complicated than **Apache** and less security issues
1. **Create a Directory to Handle Uploaded Files**
	- `sudo mkdir -p /var/www/uploads/SecretUploadDirectory`
2. **Change the Owner to www-data**
	- `sudo chown -R www-data:www-data /var/www/uploads/SecretUploadDirectory`
3. **Create Nginx Configuration File**
	- `/etc/nginx/sites-available/upload.conf`
	- ```
	  server {
	   listen 9001;
	   
	   location /SecretUploadDirectory/ {
	    root /var/www/uploads;
	    dav_methods PUT;
	   }
	  }
	  ```
4. **Symlink our Site to the sites-enabled Directory**
	- `sudo ln -s /etc/nginx/sites-available/upload.conf /etc/nginx/sites-enabled/`
5. **Start Nginx**
	- `sudo systemctl restart nginx.service`
6. **Upload File Using cURL**
	- `curl -T [file to upload] http://localhost:9001/SecretUploadDirectory/[new filename]`
