- Less invasive and more efficient than brute-forcing
- Replicates DNS records between name servers
- **"Zone transfer is a wholesale copy of all DNS records within a zone (a domain and its subdomains) from one name server to another"**

**Process**
1. **Zone Transfer Request (AXFR)**
	- Secondary DNS server initiates a zone transfer request to the primary server
	- Uses AXFR (Full Zone Transfer) type
2. **SOA Record Transfer**
	- The primary server responds by sending its **Start Of Authority (SOA)** record
	- SOA record contains vital information about the zone, including its serial number, which helps the secondary server determine if its zone data is current
3. **DNS Records Transmission**
	- Primary server then transfers all the DNS records in the zone to the secondary server, one by one
4. **Zone Transfer Complete**
	- The primary server signals end of zone transfer
5. **Acknowledgment (ACK)**
	- The secondary server sends an acknowledgment message to the primary server

**Zone Transfer Vulneraibility**
- Misconfigured DNS Server
- Reveals:
	- **Subdomains**
	- **IP Addresses**
	- **Name Server Records**

**Exploiting Zone Transfer**
- `dig axfr @[name server] [domain name]`
