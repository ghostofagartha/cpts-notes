- Web servers host multiple websites or applications on a single server, through **virtual hosting**
- Virtual hosts allow web servers to differentiate between domains, subdomains, or even separate websites with distinct content

**How Virtual Hosts Work: Understanding VHosts and Subdomains**
- **"Virtual hosting is the ability of web servers to distinguish between multiple websites or applications sharing the same IP address"**
- `HTTP` Host header, a piece of information included in every `HTTP` request sent by a web server
1. **Subdomains**
	- These are extensions of a main domain name
	- Have their own DNS records
2. **VHosts**
	- VHosts are configurations within a web server that allow multiple websites or applications to be hosted on a single server
	- Can be associated with TLD or subdomains
	- Each VHost can have seperate configuration
	- Can be configured by `/etc/hosts` file if a virtual host doesn't have a DNS record
	- **VHost Fuzzing:** Technique to discover public and non-public `subdomains` and `VHosts` by testing various hostnames against a known IP address

**Server VHost Lookup**
- The process of how a web server determines the correct content to serve based on the `Host` header:
	1. **Browser Requests a Website:** Browser initiates an HTTP request to the web server associated with that domain's IP address
	2. **Host Header Reveals the Domain:** The browser includes the domain name in the request's `Host` header, which acts as a label to inform the web server which website is being requested
	3. **Web Server Determines the Virtual Host:** The web server examines the `Host` header, and consults its virtual host configuration to find a matching entry for the requested domain name
	4. **Serving the Right Content:** The web server retrieves the corresponding files and resources and sends them back as the HTTP response

**Types of Virtual Hosting**
1. **Name-Based Virtual Hosting:** Relies solely on `HTTP Host Header` to distinguish between websites
2. **IP-Based Virtual Hosting:** Assigns unique IP address to each website hosted on the server
3. **Port-Based Virtual Hosting:** Assigns ports to each website hosted on the server
